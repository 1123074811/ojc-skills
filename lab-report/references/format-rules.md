# 实验报告格式规范

## 1. 文件命名

格式：`欧劲聪-软件23-4-20232501306-<实验报告名>.docx`

示例：
- `欧劲聪-软件23-4-20232501306-实验一 测试用例设计.docx`
- `欧劲聪-软件23-4-20232501306-实验二 缺陷管理.docx`

## 2. 封面格式

### 填写内容
- 学号：20232501306
- 姓名：欧劲聪
- 班级：软件23-4
- 日期：当日日期（格式：2026年4月14日）

### 约束
- 仅修改上述四个字段的内容
- 不修改字体、字号、间距、对齐方式等任何格式
- 保持模板的原有样式

## 3. 正文格式

### 字体与字号
| 元素 | 中文字体 | 英文/数字字体 | 字号 |
|------|----------|---------------|------|
| 正文 | 宋体 | Times New Roman | 小四（12pt） |
| 表格内容 | 宋体 | Times New Roman | 五号（10.5pt） |
| 图表题注 | 宋体 | Times New Roman | 五号（10.5pt） |

### 段落格式
- 首行缩进：2 字符
- 行距：单倍行距（或按模板默认）
- 段前/段后间距：按模板默认

### 标题间距
- 本标题结尾与下一个标题之间：最多 1 个空行
- 多余空行需删除

### 分页
- 最大标题（如"一、实验目的"）与上一段内容之间进行分页
- 使用 Word 的分页符，而非多个回车

## 4. 图表格式

### 图片格式
- 图片行居中（无缩进）
- 图片下方留空位（用户自行导入）

### 表格格式
- 表格宽度适配页面，不超出纸张边界
- 单元格内容垂直居中
- 表头可加粗

### 题注格式
- 位置：图片/表格之后一行
- 对齐：居中（无缩进）
- 格式：`图X-Y 标题` 或 `表X-Y 标题`
  - X：章节序号（如第5章则为5）
  - Y：本章内序号（从1开始）
- 字号：五号（10.5pt）

### 完整示例
```
                    [图片占位]
              图5-1 系统架构图

或：

              表5-1 用户信息表
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | int | 主键 |
```

## 5. 测试用例表格

### 表头（示例）
根据实验要求，常见表头包括：
- 测试用例编号
- 测试项目
- 测试子项
- 测试用例描述
- 前置条件
- 测试输入
- 操作步骤
- 预期结果
- 实际结果
- 测试结果

### 格式
- 字号：五号（10.5pt）
- 边框：网格型
- 对齐：表头居中，内容左对齐（或按模板）
- 宽度：适配页面，超出时自动调整列宽

## 6. 内容生成规范

### 降AIGC率改写规则

#### 动词短语扩展
- "管理" → "开展...的管理工作" / "进行管理"
- "交互" → "进行交互" / "开展交互"
- "配置" → "进行配置"
- "处理" → "去处理...工作"
- "实现" → "得以实现" / "来实现"

#### 词汇替换
| 原文 | 替换 |
|------|------|
| 采用/使用 | 运用、选用、把...当作...来使用 |
| 基于 | 鉴于、基于...来开展 |
| 利用 | 借助、运用、凭借 |
| 通过 | 借助、依靠、凭借 |
| 和/及/与 | 以及（列举时） |
| 并 | 并且、还、同时 |
| 原因 | 缘由、主要原因囊括... |
| 符合 | 契合 |
| 适合 | 适宜 |
| 特点 | 特性 |

#### 句式调整
- 使用"把"字句
- 条件句式口语化："若...则..." → "要是...那就..." / "如果...就..."
- 增加辅助词：了、的、地、所、会、可以、这个、方面、当中

#### 结构要求
- 每段内容使用"分-总-分"结构
- 字数与原文基本相符
- 不过于口语化（避免"至于xxx呢"）
- 不使用第一人称

#### 禁止修改
- 技术术语（如 Java, Python, MySQL, API 等）
- 代码片段
- 文件名、类名、函数名
- 配置项
